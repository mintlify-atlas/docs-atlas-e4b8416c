This guide walks you through generating and editing images with GPT Image 2 Skill. You will run a text-to-image command, try a single-reference style edit, combine two reference images, and inpaint a region with a mask — all from the terminal. A Python SDK example is included at the end for use in scripts and notebooks.

<Steps>

<Step title="Set your API key">

The CLI reads `OPENAI_API_KEY` from the environment or from `~/.env`. Set it once and every subsequent command picks it up automatically.

<CodeGroup>

```bash Shell export
export OPENAI_API_KEY="sk-..."
```

```bash ~/.env file
echo 'OPENAI_API_KEY=sk-...' >> ~/.env
```

</CodeGroup>

<Warning>
  Never commit your API key to version control. The `~/.env` approach keeps it out of project directories.
</Warning>

</Step>

<Step title="Generate your first image">

Run the one-liner below with `uvx` — no install needed. It calls `POST /v1/images/generations` with `model=gpt-image-2` and writes the result to disk.

```bash
uvx --from git+https://github.com/wuyoscar/gpt_image_2_skill gpt-image -p "a cat astronaut on the moon"
```

The output path is printed to stdout (for example `./2026-04-25-10-30-00-a-cat-astronaut.png`). If a `./fig/` directory exists in your working directory, the file is written there automatically.

To specify the output file, size, and quality explicitly:

```bash
gpt-image -p "a photorealistic convenience store at 10pm" \
  --size 1k --quality high -f store.png
```

The `--size 1k` shortcut expands to `1024x1024`. Use `--quality high` for sharp text, dense diagrams, and final assets; use `low` for cheap exploration drafts.

</Step>

<Step title="Edit an image with a reference">

Adding `-i` switches the endpoint to `POST /v1/images/edits`. The model receives your image alongside the prompt and produces an edited version.

```bash
gpt-image -p "Make it a winter evening with heavy snowfall" \
  -i chess.png --quality high -f chess-winter.png
```

<Tip>
  Single-reference edits work well for restyling, recoloring, translating on-image text, or applying a weather or lighting change to an existing photo.
</Tip>

</Step>

<Step title="Combine two reference images">

Pass `-i` more than once to include multiple source images in a single edits call. The model can composite subjects from different photos into one output.

```bash
gpt-image \
  -p "Place the dog from image 2 next to the woman in image 1. Match the same lighting, composition, and background. Do not change anything else." \
  -i woman.png -i dog.png \
  --size portrait --quality medium -f woman-with-dog.png
```

The `portrait` shortcut maps to `1024x1536`. Use `medium` quality for exploratory composites and `high` for final shipping assets.

</Step>

<Step title="Inpaint a region with a mask">

Supply an alpha-channel PNG mask with `-m`. Opaque pixels in the mask are preserved; transparent pixels are regenerated according to the prompt. The mask must be accompanied by at least one `-i` reference image.

```bash
gpt-image -p "replace sky with aurora" \
  -i photo.jpg -m sky_mask.png -f aurora.png
```

Create the mask in any image editor by painting the region you want to replace as fully transparent (alpha = 0) and leaving everything else fully opaque (alpha = 255).

</Step>

<Step title="Use the Python SDK directly">

For scripts, notebooks, or programmatic workflows, call the OpenAI SDK without the CLI wrapper. Use pixel-literal size strings instead of shortcut names.

<CodeGroup>

```bash CLI
gpt-image -p "Boston Spring — a botanical watercolor poster" \
  --size portrait --quality high -f poster.png
```

```python Python SDK
from openai import OpenAI

client = OpenAI()  # reads OPENAI_API_KEY from env

result = client.images.generate(
    model="gpt-image-2",
    prompt="Boston Spring — a botanical watercolor poster",
    size="1024x1536",   # portrait shortcut maps to this
    quality="high",
)

# result.data[0].b64_json contains the base64-encoded image
import base64
image_bytes = base64.b64decode(result.data[0].b64_json)
with open("poster.png", "wb") as f:
    f.write(image_bytes)
```

</CodeGroup>

For reference-based edits in Python, call `client.images.edit(...)` and pass `image=[open(p, "rb") for p in refs]` alongside the prompt. The `mask` parameter accepts an open file handle to an alpha-channel PNG.

</Step>

## Exit codes

| Code | Meaning |
|------|---------|
| `0` | Success — output path(s) printed to stdout |
| `1` | API error — full response body echoed to stderr |
| `2` | Bad arguments or missing `OPENAI_API_KEY` |
