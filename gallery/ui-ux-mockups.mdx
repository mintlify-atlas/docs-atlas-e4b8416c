GPT Image 2 produces convincing UI mockups — rendered component hierarchies, legible type, realistic iconography, and coherent layout grids. The prompts below cover three common design deliverables: a mobile fitness app screen, a dark-mode analytics dashboard, and a component library reference sheet. Each prompt specifies the design system, color values, and layout intent so the model has enough context to render a believable interface rather than a decorative approximation.

<AccordionGroup>
  <Accordion title="Mobile fitness app home screen" description="Size: portrait · Quality: high · Source: Curated">
    ```
    Design a mobile fitness app home screen mockup in light mode. Top card shows today's step count (8,432 steps, 64% of goal) with a circular progress ring. Below: three activity cards for Running, Cycling, Yoga with duration and calorie counts. Bottom tab bar with icons for Home, Workouts, Stats, Profile. Clean sans-serif typography, rounded corners, soft drop shadows, iOS-style design system, 9:16 portrait.
    ```

    ```bash CLI
    gpt-image -p "Design a mobile fitness app home screen mockup in light mode. Top card shows today's step count (8,432 steps, 64% of goal) with a circular progress ring. Below: three activity cards for Running, Cycling, Yoga with duration and calorie counts. Bottom tab bar with icons for Home, Workouts, Stats, Profile. Clean sans-serif typography, rounded corners, soft drop shadows, iOS-style design system, 9:16 portrait." \
      --size 1024x1536 \
      --quality high \
      -f fitness-app-mockup.png
    ```
  </Accordion>

<Tip>
  Specifying explicit data values — step counts, revenue figures, percentage changes — gives the model concrete text to render rather than placeholder content. Named design systems ("iOS-style", "Inter font", specific hex codes) anchor the visual language and produce more coherent results than vague style references.
</Tip>
